# GraphMaze —— 无向加权图最短路径求解器

GraphMaze 使用 Dijkstra 算法计算无向加权图中两个顶点之间的最短路径，同时附带一个批量修改边权重的工具。

## 文件列表

| 文件 | 说明 |
|---|---|
| [maze.py](maze.py) | 主程序 —— 加载图、运行 Dijkstra、输出结果。 |
| [add_one.py](add_one.py) | 工具 —— 将图中每条边的权重加上一个常数。 |
| [graph.txt](graph.txt) | 示例图文件（26 个顶点，28 条边）。 |

## 快速开始

```bash
python maze.py graph.txt A D        # 查询 A 到 D 的最短路径。
python maze.py                      # 交互模式（自动检测 graph.txt）。
```

## maze.py —— 用法

```
python maze.py [file] [start] [end] [-v]
```

**参数说明：**

| 参数 | 说明 |
|---|---|
| `file` | 图定义文件的路径。默认使用脚本所在目录下的 `graph.txt`。 |
| `start` | 起始顶点的标签。 |
| `end` | 目标顶点的标签。 |
| `-v` | 详细模式：在输出结果前打印图的摘要（顶点数和边数）。 |

若省略 `start` 和 `end`，程序将进入交互模式，列出所有可用顶点并提示输入。

**示例：**

```bash
python maze.py graph.txt A D
python maze.py my_map.txt X Y -v
python maze.py
```

## 图文件格式（`graph.txt`）

以 `#` 开头的行为注释。空行将被忽略。

### 边定义格式

```
<顶点> <顶点> <权重> [标签]
```

**示例：**

```
A B 5 ab
B C 3 bc
```

图是**无向的** —— 每条边会双向添加。可选的标签字段是附加在边上的字符串。当路径按文件中定义的相反方向经过某条边时，标签会自动反转。

### 可选指令（注释行）

```python
# vertices: A B C D
```
显式声明顶点，使孤立顶点（无边连接的顶点）也被包含在图中。

```python
# vertex_labels: A=hello B=world
```
为每个顶点附加一个标签字符串。格式：`KEY=VALUE`，以空格分隔。

边权重必须**非负**（整数或浮点数）。

## 输出格式

```
Shortest path: A → B → C → D
Total weight: 10
Concatenated string: "abbccd"
```

拼接字符串按以下规则沿路径构建：

```
vertex_label[0] + edge_label[0→1] + vertex_label[1] + ...
```

## 应用：方格迷宫

虽然 `maze.py` 操作的是抽象图，但一个常见的用例是求解**方格迷宫**中的最短路径 —— 即一个由单元格组成的网格，相邻单元格之间可能存在墙壁。你可以将这样的迷宫化简为加权图，然后直接喂给求解器。

### 化简方法

**1. 消除死胡同。** 死胡同是指只有 0 个或 1 个开口的单元格。这些走廊不可能是最优路径的一部分（除非起点或终点正好在死胡同内）。对于每个只有 1 个开口的单元格，沿着它唯一通往的方向继续移除，直到遇到一个枢纽或起点/终点为止 —— 每次移除可能暴露出新的死胡同，因此需要反复迭代直到没有死胡同为止。

**2. 标记顶点。** 死胡同消除完毕后，将**起始格**、**目标格**以及每一个**枢纽**（拥有 3 个或 4 个开口的单元格 —— 十字路口、丁字路口）标记为图的顶点。

**3. 将走廊测量为加权边。** 遍历每两个顶点之间的通道。边的权重为**"通道沿途的单元格数 + 1"**（即边同时覆盖走廊格和目标顶点格）。如果起点或终点不是枢纽，将其作为临时顶点连接到最近的枢纽。如果同一对顶点之间存在多条通道，仅保留权重最小的一条。

**4. 编写图文件。** 将结果按上述格式编码并运行 `maze.py`：

```
S A 1
A B 3
A B 1
B T 4
```

**5. 追踪最短路径。** 根据程序输出读取顶点序列。在两个顶点之间的多条通道中，始终选择单元格数最少的那条 —— 例如在上图中选择 A 和 B 之间权重为 1 的边，而非权重为 3 的边。

这样就能得到原网格中按单元格数计算的*测地线*最短路径。

### 示例

```
┌───┬───┬───┬───┐
│ S │ / │ 2   3 │
├   ┼   ┼   ┼   ┤
│ 1 | / | 1 | 4 │
├   ┼   ┼   ┼   ┤
│ A   1   B | T │
├   ┼───┼   ┼───┤
│ 1   2   3 | / │
└───┴───┴───┴───┘

```

一个带有内部墙壁的 4×2 网格。求从 S 到 T 的最短路径：
1. **消除死胡同**，包括拥有 0 或 1 个开口的单元格。对于每个只有 1 个开口的单元格，沿着它唯一通往的方向继续移除，直到遇到枢纽或起点/终点。本例中的死胡同格用 `/` 标记。
2. **将枢纽、起始格和目标格标记为顶点**。枢纽包括拥有 3 或 4 个开口的单元格。本例中的顶点用大写字母 `S`、`A`、`B`、`T` 标记。
3. **统计任意两个顶点之间通道沿途的单元格数，并以"单元格数 + 1"为权重添加一条无向边**。示例中各通道的单元格数用数字标出。
4. **将图写为输入文件**：

```
S A 1
A B 3
A B 1
B T 4
```

5. **根据程序输出追踪最短路径。** 本例的输出为 `Shortest path: S → A → B → T`。始终在两个顶点之间的多条通道中选择单元格数最少的那条，例如选择 A 和 B 之间权重为 1 的边而非权重为 3 的边。

### 原理

此化简之所以保持最短路径距离不变，是因为死胡同永远不可能包含最优路径（进出死胡同必须经过同一单元格，纯属绕路），而枢纽之间的直线通道只有一种合理的走法 —— 直接走到底。化简后的图比原网格小得多，即使在大迷宫上 Dijkstra 也能在毫秒级完成。

### 字母拼接谜题

一种常见的谜题变体是在方格迷宫的**每个单元格上写一个字母**，要求将最短路径沿途的字母拼接起来作为答案。程序的拼接字符串输出可以直接处理这种情况，只需将单元格字母编码到图文件中：

- **顶点标签**（`# vertex_labels:` 指令）—— 为每个枢纽格和起点/终点格分配标签。程序会在输出中将它们穿插在边标签之间。
- **边标签**（边定义行中可选的第四字段）—— 编码两个顶点之间走廊沿途单元格上的字母序列。如果路径按文件中定义的相反方向经过某条边，标签会自动反转 —— 因此走廊格字母必须按从第一个顶点到第二个顶点的顺序书写。

例如，给定如下化简图，其中上方一行依次拼出 `C`、`A`、`T`，下方一行拼出 `D`、`O`、`G`：

```
# vertex_labels: J1=C J2=T J3=G
J1 J2 2 AT
J1 J3 3 DO
J2 J3 4 XG
```

最短路径 `J1 → J2 → J3` 得到的拼接字符串为 `C` + `AT` + `T` + `XG` + `G` = `"CATTXGG"`。顶点标签首尾衔接边标签，拼接过程天然沿着路径方向 —— 只需走过迷宫、读取字母，程序便会自动拼出答案。

## add_one.py —— 用法

将图中每条边的权重加上一个常数。注释、空行、指令和非边行原样保留。

```
python add_one.py <input> [output] [--add N]
```

**参数说明：**

| 参数 | 说明 |
|---|---|
| `input` | 输入图文件（必需）。 |
| `output` | 输出文件。省略则输出到 stdout。可与输入文件同名以原地覆盖。 |
| `--add N` | 每条边加上的数值（默认：`1`）。可为负数。 |

**示例：**

```bash
python add_one.py graph.txt                   # 输出到 stdout。
python add_one.py graph.txt out.txt           # 写入 out.txt。
python add_one.py graph.txt graph.txt         # 原地覆盖。
python add_one.py graph.txt --add 5           # 每条边加 5。
python add_one.py graph.txt --add -2          # 每条边减 2。
```

## 算法

**Dijkstra 算法**，使用二叉堆（优先队列）。时间复杂度为 **O((V + E) log V)**，其中 V 为顶点数，E 为边数。边权重必须非负；遇到负权重时会给出警告，且可能产生错误结果。
