## 逻辑谜题 (Logic Puzzle)

### 01Sudoku — 可能性矩阵求解器
**文件：** [solver.py](../Logic%20Puzzle/01Sudoku/solver.py)

输入一个数独题目，输出一个 **27×27 的可能性矩阵**——展示每个空格在所有有效解中可能出现的每一个数字。不同于穷举所有解（当解的数量庞大时可能遗漏某些数字的放置情况），本程序通过回溯算法 + MRV 启发式策略，逐一测试每个 `(格子, 数字)` 组合的可解性。

- **输入：** 81 字符传统格式，或 729 字符 27×27 网格格式（可往返——输出可手工编辑后重新作为输入）。
- **输出：** 带分隔符的格式化 27×27 网格；终端实时进度条。
- **模板：** `--template` 标志生成全 1 模板文件，用于约束编辑。

| 文件 | 用途 |
|---|---|
| `solver.py` | 主求解器：回溯 + MRV、进度条、双格式输入输出 |
| `template_all.txt` | 预生成的 27×27 全 1 模板，供手工编辑 |
| `README.md` / `README_zh.md` | 双语文档 |

## 策略谜题 (Strategic Puzzle)

### GraphMaze — 最短路径（Dijkstra）
**文件：** [maze.py](../Strategic%20Puzzle/GraphMaze/maze.py)

使用 Dijkstra 算法 + 二叉堆（`O((V+E) log V)`）计算**无向加权图**中两个顶点之间的最短路径。支持带标签的顶点和边，可用于字母拼接类谜题变体。

- **输入：** 纯文本边行（`<顶点> <顶点> <权重> [标签]`）、顶点声明指令、顶点标签指令、`#` 注释。
- **输出：** 最短路径、总权重、拼接后的标签字符串。未指定起点/终点时进入交互模式。
- **方法论：** README 文档记载了将方形网格迷宫约化为加权图的五步方法。

| 文件 | 用途 |
|---|---|
| `maze.py` | Dijkstra 求解器：带标签的边/顶点、交互模式 + 命令行模式 |
| `example-input.txt` | 示例图：26 个顶点（A–Y），28 条带标签的边 |

### add\_one — 边权变换器
**文件：** [add_one.py](../Strategic%20Puzzle/GraphMaze/add_one.py)

读取图文件，输出一个每条边权重**增加一个常量**的副本（默认 `+1`，可通过 `--add N` 自定义；负值即为减法）。注释、空行、指令均逐字保留。支持原地覆写。

| 文件 | 用途 |
|---|---|
| `add_one.py` | 批量权重调整工具，保留所有格式 |

### Wangtu（王土）—— 国王移动逻辑谜题求解器
**文件：** [wangtu.py](../#pnku3-temp/wangtu-solver/wangtu.py)

给定一个 n×m 网格（含禁用格和国王棋子，每个国王标有所需步数），找出**所有**满足以下条件的移动方式：路径互不重叠、不踏入禁用格、最终位置国王之间不互将（切比雪夫距离 > 1）。

- **输入：** 空格分隔的网格；`#` 或 `-1` 表示禁用格，`0` 或 `.` 表示空格，正整数表示国王及其步数。
- **输出：** 完整模式显示坐标路径 + 逐步 ASCII 网格可视化；`--compact` 仅输出坐标。
- **算法：** DFS 路径枚举、回溯 + MRV 排序、增量重叠剪枝。
- **选项：** `--max-solutions N`、`--compact`、`--progress` / `--no-progress`。

| 文件 | 用途 |
|---|---|
| `wangtu.py` | 主求解器 —— 路径枚举、回溯搜索、双格式输出 |
| `example-input.txt` | 带注释的 3×3 示例谜题（两个国王） |
| `README.md` / `README_zh.md` | 双语文档 |
