# Wangtu（王图）—— 国王移动逻辑谜题求解器

给定一个 n×m 网格，其中包含禁用格和国王棋子（每个国王标有步数），找出**所有**满足以下条件的移动方式：

1. 国王之间所经过的格子（包括起点和终点）互不重叠。
2. 国王不能踏入禁用格。
3. 在最终位置，任何两个国王不能互相"将军"
   （切比雪夫距离 ≤ 1——即在 8 个方向中的任一方向上相邻）。

国王每步可向 8 个方向（水平、垂直、对角线）之一移动一格。
国王在自身路径中不能重复访问已走过的格子（自回避路径）。

| `example-input.txt` | 带注释的 3×3 示例谜题（两个 2 步国王） |

## 用法

```
python wangtu.py < input.txt
python wangtu.py puzzle.txt
python wangtu.py puzzle.txt --max-solutions 50
python wangtu.py puzzle.txt --compact
python wangtu.py puzzle.txt --no-progress
```

### 选项

| 选项 | 效果 |
|---|---|
| `--max-solutions N` | 找到 N 个解后停止（默认：1000） |
| `--compact` | 仅输出坐标；不显示逐步网格可视化 |
| `--progress` / `--no-progress` | 强制开启/关闭进度报告（默认：自动检测 TTY） |

## 输入格式

纯文本网格，空格分隔的值，每行一行。以 `#` 开头的行为注释；空行被忽略。

| 标记 | 含义 |
|---|---|
| `-1` 或 `#` | 禁用格 —— 国王不能进入 |
| `.` | 可通行的空格 |
| `0` | 0 步国王 —— 原地不动（占据起始格） |
| `1`、`2`、`3`…… | 国王，数字表示必须移动的步数 |

**示例**（`example-input.txt`）：

```
# 3×3 网格 —— 两个国王，中心为禁用格
2 . 2
. # .
. . .
```

- 国王 0 起始于 (0, 0)，需移动 2 步。
- 国王 1 起始于 (0, 2)，需移动 2 步。
- 格子 (1, 1) 为禁用格。

**含 0 步国王的示例**（`example-input-2.txt`）：

```
# 3×3 网格 —— 移动国王 + 静止国王
2 . .
. # .
. . 0
```

- 国王 0 起始于 (0, 0)，需移动 2 步。
- 国王 1 起始于 (2, 2)，**0 步** —— 原地不动。

## 输出格式

### 完整模式（默认）

每个解包括：

1. **路径列表** —— 每个国王的坐标序列。
2. **逐步 ASCII 网格** —— 显示每一步中国王的位置。

网格符号：`#` 禁用，`.` 空，`0`–`9` 表示国王 0–9，
`A`–`Z` 表示国王 10–35。

### 紧凑模式（`--compact`）

仅输出坐标序列，每个国王一行。适合机器解析。

```
=== Solution 1 ===
# King 0 (2 step(s), start r0c0)
(0,0) (0,1) (1,0)
# King 1 (2 step(s), start r0c2)
(0,2) (1,2) (2,2)
```

## 算法

1. **解析**输入网格，识别国王和禁用格。
2. **枚举**每个国王的所有有效自回避路径（DFS）。
3. **回溯搜索**，使用 MRV 排序（路径数最少的优先），找出所有满足
   路径不重叠和终点不互将约束的路径组合。
4. **输出**每个有效解，可选逐步可视化。

搜索空间逐步剪枝：一旦候选路径与已放置的国王有任何共享格子，
立即拒绝。终局互将检查仅在所有国王都放置完毕后进行。

## 依赖

Python 3.7 或更高版本。仅使用标准库 —— 无外部依赖。
