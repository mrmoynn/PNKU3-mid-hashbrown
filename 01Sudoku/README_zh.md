# 01Sudoku —— 可能性矩阵求解器

输入一个数独谜题，输出一个 **27×27 的矩阵**，展示每个原始格子在所有合法完整解中可能出现的数字。

## 27×27 映射原理

```
原始 9×9 单元格 (row, col)           27×27 区块 起始于 (row×3, col×3)
┌─────┬─────┬─────┐                  ┌──────────┬──────────┬──────────┐
│     │     │     │                  │ 1  2  3  │ 1  2  3  │ 1  2  3  │
│     │     │     │   ──────────▶    │ 4  5  6  │ 4  5  6  │ 4  5  6  │
│     │     │     │                  │ 7  8  9  │ 7  8  9  │ 7  8  9  │
├─────┼─────┼─────┤                  ├──────────┼──────────┼──────────┤
│     │  ?  │     │                  │ 1  2  3  │ 1  2  3  │ 1  2  3  │
│     │     │     │   ──────────▶    │ 4  5  6  │ 4  5  6  │ 4  5  6  │
│     │     │     │                  │ 7  8  9  │ 7  8  9  │ 7  8  9  │
├─────┼─────┼─────┤                  ├──────────┼──────────┼──────────┤
│     │     │     │                  │ 1  2  3  │ 1  2  3  │ 1  2  3  │
│     │     │     │   ──────────▶    │ 4  5  6  │ 4  5  6  │ 4  5  6  │
│     │     │     │                  │ 7  8  9  │ 7  8  9  │ 7  8  9  │
└─────┴─────┴─────┘                  └──────────┴──────────┴──────────┘
```

输出中的每个 3×3 区块对应原始谜题的一个单元格。区块内数字 1–9 按行优先排列：

```
1 2 3
4 5 6
7 8 9
```

某个数字**显示**当且仅当它在该原始单元格的至少一个合法完整解中出现过。绝不会出现的数字以 `.` 代替。

## 用法

```bash
# 从标准输入读取
python solver.py < puzzle.txt

# 从文件读取
python solver.py puzzle.txt

# 生成模板文件（所有数字均允许），用于手动编辑
python solver.py --template
```

### 输出格式

求解器始终输出带有网格分隔线的 27×27 网格（每行 35 个字符，完美对齐）：

```
123|123|123|123|123|123|123|123|123
456|456|456|456|456|456|456|456|456
789|789|789|789|789|789|789|789|789
---+---+---+---+---+---+---+---+---
123|123|...
...
===+===+===+===+===+===+===+===+===
...
```

- **数据行**宽度为 35 个字符（27 个数字 + 8 个 `|` 列分隔符）。
- **次分隔线**（`---+`）每 3 行出现一次，分隔子带。
- **主分隔线**（`===+`）每 9 行出现一次，分隔 3×3 宫带。
- 只有矩阵输出到 **stdout**；所有诊断信息输出到 **stderr**。

### 往返复用

输出可重定向到文件，手动编辑后再作为输入喂回 ——
网格绘制字符（`|`、`-`、`+`、`=`）会被自动剔除：

```bash
python solver.py puzzle.txt > result.txt
# ……编辑 result.txt，增删数字约束……
python solver.py result.txt
```

## 输入格式

支持两种格式，通过字符数**自动识别**：

| 字符数 | 格式 | 说明 |
|---|---|---|
| 81 | 传统数独字符串 | `0` / `.` 表示空格，`1`–`9` 表示已知数字 |
| 729 | 27×27 网格 | 与求解器自身输出格式相同 |

### 传统格式（81 字符）

空白字符会被忽略——可使用任意便于阅读的排版：

```
5 3 0 0 7 0 0 0 0
6 0 0 1 9 5 0 0 0
0 9 8 0 0 0 0 6 0
8 0 0 0 6 0 0 0 3
4 0 0 8 0 3 0 0 1
7 0 0 0 2 0 0 0 6
0 6 0 0 0 0 2 8 0
0 0 0 4 1 9 0 0 5
0 0 0 0 8 0 0 7 9
```

或写成一行：

```
530070000600195000098000060800060003400803001700020006060000280000419005000080079
```

### 27×27 网格格式（729 字符）

与求解器输出格式完全相同。每个 3×3 区块指定该原始单元格中**可以**出现的数字：

- 显示数字（`1`–`9`）→ 该数字在此单元格中允许出现。
- 显示点号（`.`）→ 该数字在此单元格中**不允许**出现。
- 某单元格只有一个数字 → 视为已知数（必须填该数字）。

使用 `python solver.py --template` 可生成一个起点文件（`template_all.txt`），其中全部 729 个数字均启用。

## 示例

```bash
echo "530070000600195000098000060800060003400803001700020006060000280000419005000080079" | python solver.py
```

对于具有唯一解的谜题，输出中每个 3×3 区块恰好显示一个数字（即答案数字）。对于有多解的谜题，区块可能显示多个可能数字。

## 进度条

当 stderr 为终端时，会显示实时进度条，包含已用时间和预计剩余时间：

```
[==========>           ]  45% | 245/548 | 12.3s elapsed | 14.8s ETA
```

## 工作原理

求解器不会枚举所有完整解（当解的数量庞大时，枚举法会遗漏某些可能数字），
而是逐一检查每个空格子的每个候选数字：放入该数字后，使用带 MRV（最小剩余值）
启发式策略的回溯法检测棋盘是否仍可解。

**节点限制**（每次检测最多 100,000 次递归调用）可防止求解器在病态的
"证明不可能"搜索中卡死。达到限制时，求解器保守地假设该数字*可能*出现，
并在 stderr 给出警告：

```
// Warning: search limit hit 3 time(s) — some digits may be shown as possible when they are not.
```

## 文件列表

| 文件 | 用途 |
|---|---|
| `solver.py` | 主求解脚本 |
| `template_all.txt` | 所有数字均启用的模板（由 `--template` 生成） |
| `example_easy.txt` | 示例谜题（唯一解） |
